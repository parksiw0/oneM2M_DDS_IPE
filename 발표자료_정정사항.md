# QoS_FCNT_설계.pptx 정정 사항

2026-07-15, QoS FCNT 구현(정본: `QoS_FCNT_설계서.md`)과 대조한 결과다. 발표 자료의 골격 — 토픽당 FCNT 1개, QoS 8종 = 속성, `avoid_ros_namespace_conventions` 제외, duration `"INF" | ms` 문자열, depth만 positiveInteger, "QoS마다 리소스를 쪼개면 8~9배" 논리 — 는 구현과 일치하므로 유지하면 된다. 아래는 달라진 부분만 모았다.

## 구현 확인 (발표 주장 그대로 유효)

- 토픽 하나당 FCNT 하나, 이름 `qos` — observe는 `ros2Data/<경로>/qos`, command는 `ros2Command/<경로>/qos` (슬라이드 17 구조 유지, command 방향이 추가됨)
- QoS 8종 전부 속성 매핑, avoid_ros_namespace_conventions 제외 (슬라이드 3, 16)
- deadline/lifespan/lease는 `"INF" | 십진 ms` 문자열, depth만 positiveInteger (슬라이드 9, 11, 14, 15)
- .fcp 정의 → tinyIoT `sdt_definitions/` 배포 → 재시작 → CREATE 3단계 (슬라이드 5); 배포 파일은 `config/sdt/ros-ipe.fcp`
